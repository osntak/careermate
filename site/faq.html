시작하기 전에

돈이 드나요? API 키가 필요한가요?

CareerMate 자체는 무료·오픈소스이고, 내부에 LLM이 없어 별도 API 키가 필요 없습니다. 분석·작성은 이미 쓰고 계신 AI 구독(Claude 등)이 그대로 수행하고, CareerMate는 커리어 데이터를 로컬에 저장·제공하는 “서랍장” 역할만 합니다.

어떻게 업데이트하나요?

AI에게 “CareerMate 업데이트해줘”라고 하면 최신 버전으로 올립니다(확인만 하려면 “최신이야?”). 업데이트 후에는 AI 앱 또는 MCP 연결을 재시작해 주세요.

설치 · 연결

“MCP 도구가 안 보여요” / AI가 CareerMate를 모릅니다

AI 클라이언트를 완전히 종료한 뒤 다시 시작했는지 확인하세요(설정만 바꾸고 재시작을 안 하면 적용되지 않습니다).

Claude Code는 첫 실행 시 프로젝트 서버에 대한 1회 승인을 묻습니다 — 승인하세요. 클라이언트의 /mcp 명령으로 careermate 연결을 확인하거나, AI에게 “온보딩 상태 확인해줘”라고 해 응답이 오는지 보세요.

그래도 안 되면 클라이언트별 등록 방법을 처음이세요? 또는 INSTALL.md에서 확인하세요.

ChatGPT 웹/모바일, Gemini 웹에서 안 돼요

정상입니다. 이들은 클라우드에서 실행되어 원격 URL MCP만 쓸 수 있고, CareerMate는 사용자 컴퓨터에서 도는 로컬 stdio 서버라 닿지 못합니다.

로컬에서 실행되는 Claude Desktop · Claude Code · Codex CLI를 사용하세요.

Node 버전 오류가 나요

CareerMate는 Node.js 22.5.0 이상이 필요합니다(node:sqlite 사용). node -v로 확인하고 필요하면 업데이트하세요.

대시보드

“대시보드가 안 열려요” (http://127.0.0.1:4319)

그 주소는 CareerMate가 실행 중일 때만 열립니다. AI에게 “대시보드 열어줘”라고 하면 서버가 꺼져 있을 때도 백그라운드로 자동 시작한 뒤 브라우저를 엽니다.

포트가 이미 사용 중이면 자동으로 다른 포트로 폴백합니다(CAREERMATE_PORT로 고정 가능). 자동 시작이 실패하면 ~/.careermate/server.log를 확인하세요.

대시보드는 보안상 127.0.0.1(이 컴퓨터)에만 열립니다 — 다른 기기에서는 접속되지 않습니다(의도된 동작).

브라우저를 자동으로 안 열어요

첫 온보딩 때 한 번만 자동으로 엽니다. 이후에는 링크만 안내합니다 — 직접 주소를 열거나, AI에게 “대시보드 열어줘”라고 하세요. 자동 열기를 끄려면 CAREERMATE_NO_OPEN=1.

문서(이력서·공고) 읽기

“PDF가 안 읽혀요”

스캔본/이미지 PDF(텍스트 레이어 없음)는 추출되지 않습니다. 이미지를 직접 볼 수 있는 AI라면 PDF를 그대로 보여주고, 아니면 내용을 텍스트로 붙여넣어 주세요.

암호로 보호된 PDF는 읽을 수 없습니다 — 뷰어에서 암호를 해제해 다시 저장하거나 내용을 텍스트로 붙여넣으세요. 손상된 PDF, 25MB 초과 파일도 거부되며 안내 메시지에 이유가 나옵니다.

지원하는 파일 형식은?

PDF, Word(.docx), 한컴(.hwp/.hwpx), 텍스트(.txt/.md). 가능하면 절대 경로로 파일 위치를 알려 주세요(예: C:\Users\me\Desktop\이력서.pdf).

한글이 깨져요

UTF-8과 한국어 인코딩(CP949/EUC-KR)을 자동 감지합니다. 그래도 깨지면 파일을 UTF-8로 다시 저장하거나 내용을 붙여넣어 주세요.

공고 분석 · 자소서

공고 URL은 어떻게 넣어요?

URL을 그대로 붙여넣어도 되고, 공고 본문 텍스트를 붙여넣어도 됩니다. CareerMate는 LLM이 없어 URL을 직접 크롤링하지 않으므로, 공고 본문 텍스트가 있으면 분석 품질이 가장 좋습니다.

“분석이 안 돼요” / 결과가 비어요

분석은 AI가 수행하고 CareerMate는 입력(프로필·공고)과 결과를 저장합니다. 먼저 프로필과 공고가 저장돼 있어야 합니다.

AI에게 “분석 전에 application context부터 불러와줘”라고 하면 프로필·이력서·공고를 한 번에 모아 분석 맥락을 갖춥니다.

면접 준비는 언제 만들 수 있나요

공고가 저장되어 있으면 AI에게 면접 준비를 요청해 저장할 수 있습니다. 지원 상태가 ‘서류 합격’ 이상이 되면 대시보드가 면접 준비를 다음 행동으로 더 적극적으로 제안합니다.

이력서·자소서를 파일로 받을 수 있나요?

네. AI에게 “이력서 파일로 내보내줘”(또는 자소서/프로필)라고 하면 ~/.careermate/exports에 Markdown 또는 인쇄용 HTML(브라우저에서 “인쇄 → PDF로 저장”)로 저장됩니다.

대시보드 문서·프로필 페이지에서 버튼으로 직접 다운로드할 수도 있습니다.

데이터 · 프라이버시

내 데이터는 어디에 저장되나요?

사용자 컴퓨터의 ~/.careermate(Windows: %USERPROFILE%\.careermate)에만 저장됩니다. 대시보드 설정 > 데이터 저장 위치에서 정확한 경로를 볼 수 있고, 외부로 전송되지 않습니다.

백업하거나 전부 삭제하려면?

대시보드 설정 > 백업/내보내기에서 백업 생성·전체 JSON 내보내기를, 위험 구역에서 전체 초기화를 할 수 있습니다(초기화 직전 자동 백업).

여기서 해결되지 않으면 GitHub 이슈에 남기거나 osntak@gmail.com으로 문의하세요. 보안 문제는 이슈가 아니라 SECURITY.md의 비공개 절차를 이용해 주세요.
