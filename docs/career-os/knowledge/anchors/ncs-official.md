# NCS 공식 앵커 (경력기술서)

## 사용 규칙 (binding)

이 앵커는 `career-description.md`와 `eop/career-description.md`를 보강하는 NCS 공식 데이터다.
규칙은 **보강(override 아님)** 이다 — 기존 EOP·플레이북과 충돌 시 기존 문서 우선.

- 경력 vs 경험 분류는 반드시 '금전적 보수 수령 여부'를 기준으로 한다. 보수 여부가 불명확한 경우 사용자에게 확인하고 추측으로 분류하지 않는다.
- 신입/무경력 작성 시 경력사항이 비어 있으면 경험사항(경험기술서)으로 전환하되, 경력을 지어내지 않는다.
- 직무기술서가 제시된 경우 그 안의 필요지식·필요기술·직무수행내용·직업기초능력 항목을 우선 키워드 소스로 삼는다.
- 채용절차법 제4조의3 금지 정보(신체 조건·출신지역·혼인여부·재산·가족 학력·직업·재산)는 경력기술서 본문에 어떤 형태로도 포함하지 않는다.

## 데이터 (확인된 [1차]만)

### 1. 경력(경력사항) vs 경험(경험사항) 정의

**출처:** ncs.go.kr FAQ, moel.go.kr NCS 기반 입사지원서, alio.go.kr 블라인드 채용 입사지원서

| 구분 | 정의 | 예시 |
|------|------|------|
| **경력(사항)** | 금전적 보수를 받고 일정 기간 수행한 직무 관련 이력 | 정규직·계약직·파트타임·인턴(급여 지급) |
| **경험(사항)** | 금전적 보수 없이 수행한 직무 관련 활동 | 동아리·대외활동·봉사·팀 프로젝트·교육과정 참여 |

> **주의:** 인턴이라도 급여를 받았으면 경력사항에 기재한다. 동일 활동이라도 보수 유무에 따라 섹션이 달라진다. NCS 입사지원서에서 이 구분은 기계적 분류 기준이므로 추측하지 말고 사용자에게 확인한다.

### 2. NCS 블라인드 표준 입사지원서 섹션 구조

**출처:** ncs.go.kr 블라인드채용 안내(RH-202-003-02), moel.go.kr 표준 입사지원서 양식

NCS 블라인드 표준지원서는 아래 5개 섹션으로 구성된다. 고용 기회에 영향을 줄 수 있는 개인 식별 정보(출신학교·사진·신체조건 등)는 배제하고, **직무 관련 능력 증거** 중심으로 구성된다.

1. **인적사항** — 이름·연락처 등 최소 식별 정보. 출신지역·가족사항·사진·신체조건 등 직무 무관 항목은 배제.
2. **교육사항** — 직무 관련 교육·훈련 이력.
3. **자격사항** — 직무 관련 국가기술자격·민간자격 등.
4. **경력사항** — 금전적 보수를 받고 수행한 직무 관련 이력 (위 §1 정의).
5. **경험사항** — 보수 없이 수행한 직무 관련 활동 (위 §1 정의).

> **비고:** '우대 및 결격사항' 등 추가 항목이 기관별로 붙을 수 있다. 위 5개가 NCS 표준 공통 섹션이다.

### 3. 직업기초능력 10개

**출처:** ncs.go.kr 직업기초능력 페이지(TH0302List.do?dirSeq=121) — 직접 확인

직종·직위에 무관하게 모든 직업인에게 공통으로 요구되는 기초 역량. NCS 채용 직무기술서에서 평가 항목으로 명시되며, 경력기술서·자기소개서에서 역량 키워드 소스로 활용한다.

| 번호 | 능력명 |
|------|--------|
| 1 | 의사소통능력 |
| 2 | 수리능력 |
| 3 | 문제해결능력 |
| 4 | 자기개발능력 |
| 5 | 자원관리능력 |
| 6 | 대인관계능력 |
| 7 | 정보능력 |
| 8 | 기술능력 |
| 9 | 조직이해능력 |
| 10 | 직업윤리 |

> **사용 지침:** 직무기술서에 이 중 특정 항목이 명시된 경우, 경력기술서에서 해당 역량을 증거(행동·산출물·결과)로 뒷받침한다. 역량명을 그대로 나열하지 말고 구체적인 사례로 입증한다.

### 4. NCS 채용 직무기술서 표준 필드

**출처:** ncs.go.kr 검색 결과 다수 및 공개된 직무기술서 문서 제목·구조 일관 확인. PDF 직접 텍스트 추출 불가 — 아래는 검색 결과에서 반복 확인된 필드명이며, 필드 총 개수는 단정하지 않는다.

확인된 필드:

- **채용분야** — 모집 직무명 (예: 일반행정, IT개발)
- **분류체계** — 대분류 / 중분류 / 소분류 / 세분류(직무)
- **직무수행내용** — 해당 직무에서 실제 수행하는 업무 내용
- **능력단위** — NCS 능력단위 번호 및 명칭 (직무별 세부 단위)
- **필요지식** — 직무 수행에 필요한 이론·법규·개념 지식
- **필요기술** — 직무 수행에 필요한 실무 기술
- **직무수행태도** — 직무 수행 시 요구되는 태도·가치관
- **관련자격** — 직무 관련 국가기술자격 및 민간자격
- **직업기초능력** — 위 §3에서 정의된 10개 중 해당 직무에 요구되는 항목

> **사용 지침:** 공고에 직무기술서가 첨부된 경우, '필요지식'·'필요기술'·'능력단위'를 1차 키워드 소스로 삼는다. '직무수행내용'은 경력기술서의 역할·성과 서술과 정렬 기준이 된다. 키워드는 역할·성과 문장 안에 자연스럽게 녹이고 나열하지 않는다.

### 5. 채용절차법 직무 무관 개인정보 수집 금지

**출처:** 채용절차의 공정화에 관한 법률 제4조의3(출신지역 등 개인정보 요구 금지) — **본조 2019년 신설(법률 제16321호)**. law.go.kr·casenote.kr 확인. (sources의 law.go.kr URL `(20200526,17326,…)`은 조문을 열람한 2020-05-26 개정본 버전 스냅샷이며, 제4조의3 자체의 신설 근거 법률번호가 아니다.)

구인자는 기초심사자료(지원서)에 아래 정보를 기재하도록 요구하거나 입증자료로 수집할 수 없다.

| 금지 범주 | 구체 항목 |
|-----------|-----------|
| 신체적 조건 | 구직자 본인의 용모·키·체중 등 |
| 출신 관련 | 구직자 본인의 출신지역·혼인여부·재산 |
| 가족 관련 | 직계 존비속·형제자매의 학력·직업·재산 |

> **적용:** 경력기술서 작성 시 위 항목을 본문에 포함하지 않는다. 자사 양식이 강제적으로 요구하는 경우는 `ats.md §개인정보 배제`의 '양식 우선' 분기를 따른다. 위반 시 과태료(500만 원 이하).

## 검증 보류 (자산 아님 — 후속 출처검증 과제)

아래 항목은 수집 단계에서 파인더가 발견했으나, 공식(1차) 출처 직접 확인이 안 됐거나 벤더·커뮤니티 블로그 2차 인용에만 존재한다. 현 라운드에서 **자산화하지 않는다.** 후속 라운드에서 공식 출처로 확인된 경우에만 이 파일에 이동한다.

- **[2차] Wanted "정량화 이력서" 합격률 +13%p / 합격 이력서 정량표현 36%↑** — 벨로그(velog.io) 등 커뮤니티 블로그에서 인용되나, Wanted 공식 블로그 또는 1차 데이터 리포트 원문 직접 확인 불가. 수치 자체가 마케팅 자료일 가능성. 자산화 금지.
- **[2차] 직무별 winning 키워드(개발자·PM, Wanted 3M 이력서 샘플)** — Wanted 블로그(공식 게시물 존재 가능성 있음)에서 인용되나 직접 접근·원문 텍스트 확인 미완. 후속 검증 시 Wanted 공식 블로그 URL을 직접 fetch해 원문 인용 여부 확인 필요.
- **[2차] Searchright 3-템플릿 분류(역순연대기식·직무중심식·혼합형)** — Searchright는 커리어 컨설팅 벤더. 분류 자체는 업계 통용 개념이나 특정 템플릿명·필드 구조는 벤더 독자 프레이밍. 공식 표준 아님.
- **[2차] 경력기술서 분량 기준(연차별 권장 페이지·글자수)** — 한국 경력기술서 특화 공식 기준 없음. 사람인·잡코리아 가이드라인이 존재하나 플랫폼별 상이하고 공식 표준 아님. resume.md의 국문 분량 기준(≤2,000자/≤4,000자)은 한국 공식 기준이 아닌 실무 관행 추정임을 주의.
