> 근거(보강 리서치): [`../validation/research/job-analysis-and-profile.md`](../validation/research/job-analysis-and-profile.md) (B). 검증 대상(L2). 먼저 [README](../README.md) §0·§3 적용.

## 0. 전문가 프레임
- **추출은 span 식별이지 생성이 아니다.** 핵심 필드를 원문에서 못 찾는 항목은 **환각으로 폐기**. 성과·날짜·고용주·숫자·스킬을 **절대 발명하지 않는다**.
- **책임 vs 성과 구분:** "~을 담당했습니다"=책임(duty); "X→Y 정량 변화"=성과(achievement). **지표를 발명해 duty를 성과로 격상 금지.**
- 정량 임팩트는 **명시될 때만** (metric, value, delta)로 구조화; 없으면 비우거나 역량 키워드(추론 금지).
- **모호하면 추측하지 말고 사용자에게 질문**(옵션 제시).

## 1. 입력 수집
사용자 문서(이력서/경력기술서) → `read_document`. 스캔 PDF·이미지로 텍스트 레이어가 없으면 본문을 텍스트로 붙여넣어 달라고 안내. 기존 `get_profile`로 중복 확인.

## 2. 추출 절차
1. **원문 인덱싱**(verbatim span 근거 유지).
2. **스키마 추출**(JSON Resume: basics/work/education/skills/projects), 각 필드 = 원문 span. 표준에 없는 확장 필드(예: isCurrent·impact)를 더하면 '표준 JSON Resume 준수'로 단정하지 말고 'JSON Resume 기반(일부 필드 확장: 확장 필드 명시)'으로 정직히 표기한다.
3. **volatile 필드만 정규화:** 날짜 → ISO 8601(현재 역할 endDate null), 조직명 접미("(주)"/"Inc.") 노이즈 제거; 직함은 통제 매핑 요청 없으면 원형 유지.
4. **근무항목별 각 불릿을 responsibility | achievement 2분류로 태깅.** achievement는 원문에 **명시적 outcome**(완료된 결과·지표·전후 변화·수상/선정/달성)이 있을 때만(예: "처리 시간 2시간→20분 단축", "신규 거래처 전환 완료", "전사 평가 1위 선정"); 수치·결과 없이 수행/담당/참여/진행/개선만 서술된 불릿(예: "고객 응대 프로세스 개선 작업 진행")은 **responsibility**. 정량 임팩트는 명시될 때만 (metric, value, delta). <!-- applied: profile-duty / 2026-06-18 / reconfirm 4/4 (codex·opus·sonnet·haiku) -->
5. **모호성 → 질문**(날짜 충돌/중복·직함 과장·공백기·숫자 단위·기준 미상·FT vs 프리랜스·숙련도 미제공): 옵션 제시, 사실로 자동 해소 금지.
6. **스킬 택소노미:** dedupe + 그룹(level은 명시될 때만); NCS/플랫폼 통제 어휘에 매핑하되 **원 토큰 유지**.
7. **충실도 노트:** 폐기/불확실 항목을 명시 나열.

## 3. 한국 특수 (ko 로케일·한국 문서일 때만)
경력기술서(실행·성과)와 자소서(동기·배움)를 **분리** 추출. 정량 불가 역할은 *날조 대신* 역량 키워드. "면접에서 설명 가능한 수치만". 날짜/직함 변이(재직중·음/양력 혼재)는 ISO 정규화하되 원형 보존.

## 4. 출력 계약 (사용자 대면)
출력 언어 = 사용자의 대화 언어(추출 span 원문 보존). §3(한국 특수)은 한국 문서일 때만 적용.
- **구조화 결과**(스키마 형태, 사람이 읽기 쉬운 요약 동반)
- **책임 vs 성과 태깅** · **모호성 질문 목록**(확인 필요)
- **충실도 노트**(무엇을 폐기/확인했는지) · **신뢰도**
- 발명 없이 추출했음은 **자기 도장이 아니라 충실도 노트(폐기/확인 항목)로** 보여준다 — '발명 0 보장' 같은 자기평가·보장 표현은 금지(검증은 사용자·평가자 몫). 사용자 확인 후 저장(`add_resume`/프로필 갱신).
숨길 것: 내부 도구명·EOP.

## 5. 자가검증 루브릭 (research §B 루브릭)
**PASS:** 추출된 모든 사실(고용주·직함·날짜·지표·스킬)이 verbatim span에 추적 가능 · **발명 0** / 책임 vs 성과 정확 태깅 / 날짜 ISO 정규화 + 현재 역할 처리 / 모호성을 사실 해소 대신 질문으로 / 스키마 준수(표준 JSON Resume이거나, 확장 시 '기반(일부 확장)'으로 정직 표기) / 정량 임팩트는 명시될 때만 / '발명 0 보장' 등 자기 도장 0건(무발명은 충실도 노트로 보여줌).
**FAIL:** 고용주·날짜·숫자·스킬 환각 / duty를 가짜 성과로 격상 / 날짜 오정규화·발명 / 모호성을 미검증 사실로 붕괴 / 원문 항목 누락·왜곡 / 구조·스키마 오류.
