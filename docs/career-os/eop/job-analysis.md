# EOP — 공고 분석·분류 (Job Posting Analysis & Classification)

> 외부 LLM 실행 지침. 이 절차를 따르면 키워드 스크래핑이 아니라 **채용 분석가**처럼 공고의 행간을 읽는다.
> **지식 출처(deep reference, 먼저 읽어라):** [`../knowledge/fit-matching.md`](../knowledge/fit-matching.md) · [`../knowledge/ats.md`](../knowledge/ats.md) · [`../knowledge/company-research.md`](../knowledge/company-research.md). 이 EOP는 그 플레이북을 *실행 절차*로 압축한 얇은 층이다 — 분석 시 함께 적용한다.
> 근거(보강 리서치): [`../validation/research/job-analysis-and-profile.md`](../validation/research/job-analysis-and-profile.md) (A). 검증 대상(L2). 먼저 [README](../README.md) §0·§3 적용.

## 0. 전문가 프레임
- **스크랩 말고 디코드.** 필수 vs 우대는 소프트 위계이며, 우대가 *팀이 진짜 원하는 것*을 더 드러내기도 한다.
- **우선순위 = 위치 + 반복**(상단·교차 반복 = 입장료). 핵심 요건 70~80% 매치면 충분.
- **시니어리티는 동사에 인코딩**(총괄·주도·owns vs 보조·지원·assists), 직함과 교차확인.
- **행간 신호**(팀 단계·긴급도·스코프 모호·문화 톤)와 **레드플래그**(모호 동사만, "멀티플레이어", self-starter+무지원, 급여밴드/팀정보 없음, **열정페이**)를 *트리거 문구 인용*과 함께.
- **증거(JD 인용)와 추론(라벨링)을 분리** — 추론을 사실로 제시하지 않는다.
- **확신도 분리:** 원문 인용=확정, 그 외는 '추정'이며 확신도를 낮춰 표현한다(예: "이메일 접수형"은 확정, "수시 채용"은 낮은 확신의 추론). **외부 분류 코드(NCS 번호 등)는 데이터로 확정될 때만** 쓰고 아니면 계열명만(코드 발명 금지). 산출물에 'EOP'·내부 기준명을 노출하지 않는다.

## 1. 입력 수집
공고 텍스트(`get_application_context`의 job, 또는 사용자가 준 경로를 `read_document`/본문 붙여넣기). 회사·플랫폼 맥락.

## 2. 분석 절차
1. **세그먼트:** 직무내용 / 자격요건(필수) / 우대사항 / 처우·복지 / 회사소개.
2. **원자 요건 추출 + must/preferred 태깅**(섹션 + 반복 + 모달 "필수/required/must" vs "우대/preferred").
3. **우선순위 채점**(빈도 + 위치; 상단+교차반복=높음).
4. **시니어리티 추론**(동사 레지스터 + 연수) + 직함 교차확인으로 스코프 미스매치 탐지.
5. **암묵 신호 표면화:** 팀 단계·긴급("즉시 전력")·스코프 모호·문화 톤.
6. **레드플래그 플래그**(트리거 문구 인용).
7. **분류:** NCS 직무군(대/중/소/세분류) + 레벨(entry/mid/senior/lead) + 채용유형(공채/수시/인턴 채용형·체험형) + 플랫폼 카테고리.
8. **증거 vs 추론 분리.**

## 3. 한국 특수
우대 = 가점이나 반복·강조·시니어 역할이면 *사실상 필수*로 격상. NCS 직무기술서 = 공공 표준 JD(능력단위). 공채(연 1~2회 배치) vs 수시(롤링·현재 지배적). **열정페이**(경험 포장 저임금)는 명시 플래그. 직무무관 정보(나이·사진·가족 등)를 요구하면 채용절차법 위반 가능으로 플래그.

## 4. 출력 계약 (사용자 대면)
- **핵심 요약**(직무·레벨·채용유형 한눈에)
- **필수 vs 우대 분리 표** + 우선순위
- **시니어리티·팀 단계 추론**("추정"으로 명시, 증거 인용)
- **레드플래그**(트리거 인용)
- **분류**(NCS + 플랫폼)
- **다음 단계**(적합도 분석/자소서로 연결) · **신뢰도**
숨길 것: 내부 도구명·EOP.

## 5. 자가검증 루브릭 (research §A 루브릭)
**PASS:** must vs preferred 정확 분리(평면 키워드 리스트 아님) / 암묵 신호 ≥1개를 인용 증거와 식별 / 진짜 레드플래그 플래그 / 직무군+레벨+채용유형 정확 분류 / 추론을 인용 증거와 분리.
**FAIL:** 모든 스킬을 동등 필수로(스크랩) / 텍스트에 없는 요건 발명 / 시니어리티 오독(동사 무시) / 명백 레드플래그 누락(열정페이·무지원) / 분류 틀림·없음 / 추론을 사실로 제시.
