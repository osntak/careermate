# EOP — 면접 준비 (Interview Prep)

> 외부 LLM 실행 지침. 이 절차를 따르면 질문 나열이 아니라 **현업 면접관(행동+기술)** 수준의 준비 킷이 나온다.
> **지식 출처(deep reference, 먼저 읽어라):** [`../knowledge/interview-behavioral.md`](../knowledge/interview-behavioral.md) · [`../knowledge/interview-technical.md`](../knowledge/interview-technical.md) · [`../knowledge/company-research.md`](../knowledge/company-research.md). 이 EOP는 그 플레이북을 *실행 절차*로 압축한 얇은 층이다 — 준비 시 해당 knowledge 문서를 함께 적용한다.
> 근거(보강 리서치): [`../validation/research/interview-prep.md`](../validation/research/interview-prep.md). 검증 대상(L2). 먼저 [README](../README.md) §0·§3 적용.

## 0. 전문가 프레임
- **구조화가 카리스마를 이긴다(Google):** 모든 예상질문을 정의된 역량 + 빈약/경계/양호/탁월 기준에 매핑한다.
- **역량-우선(질문-우선 아님):** 스토리뱅크 → 스토리×역량(×인재상) 매트릭스 → 거기서 질문 생성.
- 외운 이론보다 **체험·구체+숫자.** 저장 데이터의 실제 증거를 강제하고 사실을 날조하지 않는다.
- **한국 이중축:** 직무적합 + 조직적합(인재상) + 인성. 실무면접=직무경험, 임원면접=인성+회사이해+인재상.
- 면접 준비 저장은 **'서류 합격'(document_passed) 이상에서만 해금**(상태 게이팅 존중).
- **답변 = 내 경험(앵커 필수) + 도메인 지식(개념/제안으로 분리).** '제가 했다'로 말할 수 있는 건 저장 데이터뿐이다. 직군별 전문 개념(예: 개발=격리수준·idempotency, 마케팅=어트리뷰션·LTV, 재무·회계=감가상각·헤지, 생산·품질=공정능력·관리도)은 '개념 설명' 또는 '입사 후 제안'으로 제시하되 *실제 한 일처럼 단정 금지*. **통계·예측타당도·측정 환경(운영 대시보드 등)·내부 지표·상황 묘사(예: 온콜·성수기 현장)는 출처에 없으면 발명 절대 금지**(README §0 사실 3분류). 미확인 항목은 답변 본문이 아니라 '면접 전 확인' 별도 섹션으로.

## 1. 입력 수집
`get_application_context` → profile·경력·대상 job·기존 면접 자료. 지원 상태 확인(document_passed↑인지). **역할군**(eng/PM/design/data/sales/기타)·**면접 단계**(실무/임원)·**형식**(인성/PT/토론/압박) 판정.

## 2. 분석·작성 절차
1. **스토리뱅크:** 갈등·실패·리더십/주도·모호함·무권한 영향·압박하 delivery 등 실제 경험 6~10개를 저장 데이터에서 인벤토리.
2. **역량/인재상 매핑:** 각 스토리를 JD 역량 + 회사 인재상에 태깅(매트릭스), flex 변주 준비.
3. **예상질문 생성:** P2 분류(직무지식/문제해결/리더십)별 + 한국 형식별. 각 질문에 **루브릭 라인(빈약/경계/양호/탁월)**.
4. **답변 골격:** STAR(S/T 짧게, **A 우세·1인칭·본인 기여 명시**, R 정량+교훈), 답변당 ~60~90초.
5. **꼬리질문 2~3개 + 준비 답변**(전문가 차별점: "왜 그 방법?/팀 vs 본인?/다시 한다면?/측정 결과?"). — "왜/어떻게/어떤 순서로" 답변에서 의사결정 이유·측정 근거·진행 순서가 저장 데이터에 있으면 1인칭으로 말하고, 없으면 후보의 실제 판단으로 단정하지 말고 "면접 전 확인"으로 분리한다(도메인 일반 원칙은 "일반적으로는 ~", 1인칭 행동성은 유지). <!-- applied: C4 interview-why / 2026-06-18 / reconfirm 4/4 (codex·opus·sonnet·haiku) -->
6. **난질문 사전 스크립트:** 실패/약점(*진짜* 약점+진행 중 완화책), 압박(차분 조목조목 Yes-But).
7. **역질문 3~5개**(구체·역할 현실적: 온보딩·담당 범위·성공 기준).
8. **역할별 기술:** 시스템디자인 3단계+트레이드오프 / PM CIRCLES+성공지표 / 데이터 SQL+케이스 / 영업 파이프라인+방법론 / KR PT(논리·실행가능성)·토론(입장+근거 차분).

## 3. 한국 특수 (ko 로케일·한국 면접일 때만 — 인재상·압박면접·잡플래닛/Blind는 비한국 면접에 적용하지 않는다)
실무 vs 임원 단계별 가중 조정. 인재상은 *본인 언어*로 패러프레이즈 + 개인 가치 연결(암기 금지). 압박면접 빈출(좋아하는 일 vs 잘하는 일·공백기·부당 지시·안 맞으면·중복 업무) 사전 대비. 커뮤니티 기출(잡플래닛/Blind)은 방향성 참고만(검증).

## 4. 출력 계약 (사용자 대면)
출력 언어 = 사용자의 대화 언어. §3(한국 특수)은 한국 면접일 때만 적용.
- **면접 준비 킷:** 예상질문 + 모범 답변 골격 + 질문별 루브릭 라인 + 꼬리질문/준비답변
- **스토리×역량 매트릭스** · **역질문 목록** · **난질문 대비**(실패·약점·압박)
- 결론 / 근거(저장 경험 인용) / 리스크·한계 / 신뢰도 / 다음 단계
숨길 것: 내부 루브릭·도구명·EOP. 저장=`save_interview_prep`(document_passed↑).

## 5. 자가검증 루브릭 (research §리뷰어 — **R1·R2·R5 중 FAIL이면 전체 FAIL**)
R1 증거기반·비제너릭(후보의 실제 스토리/숫자) / R2 역량·인재상 매핑(매트릭스) / R3 STAR 품질(A 우세·1인칭·R 정량+교훈) / R4 꼬리질문 깊이(질문당 2~3) / R5 난질문 처리(실패·약점=인정+성장, 압박 사전스크립트, 레드플래그 없음) / R6 역할별 기술 프레임+결정 강조 / R7 역질문 3~5 / R8 슬롭·환각 없음.
**통과 = R1·R2·R5 PASS + 나머지 양호 + 모든 답변이 저장 데이터에 앵커.**
