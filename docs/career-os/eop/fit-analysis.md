# EOP — 적합도 판정 (Fit Analysis)

> 외부 LLM 실행 지침. 이 절차를 따르면 키워드 매칭이 아니라 **시니어 리크루터/헤드헌터 수준**의 적합도 분석이 나온다.
> **지식 출처(deep reference, 먼저 읽어라):** [`../knowledge/fit-matching.md`](../knowledge/fit-matching.md) · [`../knowledge/ats.md`](../knowledge/ats.md) · 검증기 [`../knowledge/verifiers/truthfulness.md`](../knowledge/verifiers/truthfulness.md)·[`consistency.md`](../knowledge/verifiers/consistency.md). 이 EOP는 그 플레이북을 *실행 절차*로 압축한 얇은 층이다 — 분석 시 해당 knowledge 문서의 원칙·루브릭을 함께 적용한다.
> 근거(보강 리서치): [`../validation/research/fit-analysis.md`](../validation/research/fit-analysis.md). 검증 대상(PROTOCOL L2). 먼저 [README](../README.md) §0 근거 우선순위·§3 출력 계약을 적용.

## 0. 전문가 프레임 (이렇게 생각하라)
- 적합도는 키워드 매칭이 아니라 **관련 경험의 행동 증거**로 결정된다(SHRM: 스크리닝에서 경험>스킬>학력). 키워드는 보조 증거일 뿐 결론이 아니다.
- 채점 *전에* **MUST(녹아웃) vs 우대(가점)** 를 분리한다. 진짜 녹아웃 = "없으면 직무 수행/법적 자격이 불가능". 테스트: "그게 없어도 탁월하면 뽑겠다"면 녹아웃 아님. 녹아웃은 ~3–5개로 제한.
- 녹아웃=이진(충족/미충족/확인필요), 우대=그라데이션 가점이며 **부재해도 감점 아님**.
- 전이가능 스킬은 **자기주장이 아니라 행동 증거**로만 인정.
- 과적합(overqualified)은 자동 탈락이 아니라 **리스크+완화질문**으로 표시.
- 전역 직감 점수 금지 — **요건별로 증거를 인용하고 강도 등급**을 매긴다(구조화 채점이 인상 채점을 이긴다).
- 높은 키워드 커버리지를 높은 적합으로 등치하지 않는다(>75~80%는 스터핑 신호).

## 1. 입력 수집
`get_application_context`를 먼저 호출 → profile·대표 resume·경력/스킬·대상 job(requirements/keywords/직무내용). 공고가 파일/URL이면 사용자가 준 경로를 `read_document`로 읽거나 본문을 받는다. 부족한 정보는 지어내지 말고 사용자에게 확인.

## 2. 분석 절차 (순서 고정)
1. **JD 원자 분해** → 3버킷: 필수/MUST(녹아웃) · 우대/PREFERRED(가점) · 맥락/CONTEXT(직무·환경, 관련성 판단용). 각 요건은 단일 검증가능 주장.
2. **녹아웃 먼저 게이트.** 각 MUST = 충족/미충족/확인필요. 하드 MUST 하나라도 미충족(법적 자격·필수 자격증·진짜 시니어의 "N년 이상") → 나머지와 무관하게 최종 **낮음/FAIL**, 어떤 녹아웃인지 명시. 녹아웃이 확인필요 → 통과 불가, blocking missing-info.
3. **요건별 증거 매핑(핵심).** 각 요건 ← 정확한 resume 증거(프로젝트·역할·지표·맥락 내 도구) → 강도: **Strong**(했음+성과/지표) / **Moderate**(인접·전이) / **Weak**(언급, 증거無) / **None**. 키워드만 있고 행동 증거 없으면 **Weak**(매치 아님).
4. **가중 채점(평균 아님).** MUST는 게이트. 충족분 중 관련경험 > 맥락 내 하드스킬 > 자격 > 우대 순 가중. 우대는 상방만 더하고 부재는 무감점. 밴드(높음/중간/낮음 또는 0–100)를 추론과 함께 제시.
5. **리스크/레드플래그를 점수와 분리.** 과적합·미설명 공백기·짧은 근속·스코프 미스매치(IC가 리드 지원, 리드 증거無)·도메인 미스매치 → **면접 확인 항목**(감점 아님, 진짜 녹아웃 제외).
6. **미싱인포 정직 선언.** resume가 침묵하는 요건 = "확인 필요"(가정된 통과/탈락 금지). 녹아웃급 미상은 verdict 보류, 우대급 미상은 신뢰도만 하향.

## 3. 한국 시장 특수
- 자격요건 우산 안의 **필수**(자격증·어학·법적 요건)가 진짜 녹아웃. **우대**=가점이나 반복·강조·시니어 역할이면 *사실상 필수*로 격상해 취급.
- **신입 공고에 경력연수 녹아웃 적용 금지** — 잠재력·기초·직무관련 교육/경험으로 평가. 경력은 도구+지표 기반 즉시전력으로 평가.
- 동사 레지스터로 레벨 추론(신입 지원·수행·참여 → mid 운영·분석·기획 → senior 전략수립·리드·의사결정), 직함과 교차확인해 스코프 미스매치 탐지.
- **공채/NCS·블라인드 맥락:** 직무기술서의 능력단위(지식·기술·태도·경험)에 증거 매핑. **학력·출신·나이·외모·가족 신호 사용 금지**(채용절차법 제4조의3 / 공정채용). JD가 그런 직무무관 정보를 요구하면 사용자에게 플래그.

## 4. 출력 계약 (사용자 대면 — 전부 포함)
1. **종합 점수 + 밴드** (한 줄 근거, MUST+가중 증거 기반; 키워드% 아님)
2. **녹아웃 패널** — 각 MUST: 충족/미충족/확인필요 (실패·미상 명시)
3. **요건별 증거 표** — 요건 → resume 증거 인용/위치 → 강도 → 전이 크레딧 주석
4. **갭** — 필수-부재/약한 항목, 임팩트순
5. **리스크/레드플래그** — 면접에서 확인할 항목으로 표현
6. **미싱인포** — 정직하게(녹아웃급 미상=blocking)
7. **신뢰도** — 미상 많을수록 하향
8. **다음 단계** — 갭/리스크별 구체 액션(면접에서 무엇을 확인·보완할지)

내부 도구명·이 EOP·점수 계산 내부는 노출하지 않는다.

## 5. 자가검증 루브릭 (제출 전 스스로 통과 확인 — research §F)
**즉시 FAIL(얕음) — 하나라도:** 증거 매핑 없는 키워드 카운트 fit% / MUST·우대 미분리 또는 우대로 탈락 / 높은 커버리지=높은 적합 / 증거 환각·미상 가정 / 학력·출신·나이·외모 등 금지 신호 사용 / 과적합 자동탈락·신입에 경력 녹아웃 / 요건별 증거 없는 인상·단일 직감 점수.
**PASS(전문가) — 전부:** JD 원자분해+버킷 / 녹아웃 선게이트+명시 / 요건별 증거 강도+전이 근거(행동 증거) / 관련경험 가중·우대 상방만 / 리스크 분리 / 미싱인포 정직+신뢰도 조정 / 한국 관행(필수·우대·신입/경력·NCS·금지 신호) 준수.
**탁월(가점):** ATS가 놓치는 동의어/도구 등가 포착 / 사실상 필수 우대 식별 / 갭·리스크별 구체 next action.
