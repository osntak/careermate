# 지식 보강 시도 → 기각 (negative result, 2026-06)

> 보조 문서(런타임 소비 경로 아님). **결론: 앵커데이터 자산을 만들어 `get_playbook`에 배선했으나, A/B 검증에서 효용이 입증되지 않고 1건 회귀가 나와 되돌렸다.** 검증 루프 R8~R15 교훈("prose 코퍼스는 이미 완성 — 추가는 무용, 때로 회귀")을 **prospective(사전적)하게 재확인**한 사례. 같은 시도를 반복하지 않기 위한 기록.

## 0. TL;DR

- **요청:** "검증된 정보를 최대한 찾아 EOP/지식에 통합해 LLM 능력을 향상."
- **리서치 결론(멀티에이전트, 4도메인×5앵글, 적대적 출처검증):** 산문(지침) 추가할 것 **0건**, 중복 다수. 유일한 후보 레버 = **구조화 앵커데이터**(문항 디렉티브·시니어리티 동사·녹아웃 택소노미·전이스킬·글자수 모드·역량 enum·NCS 공식).
- 앵커 **8종**을 만들어 `get_playbook` 자동첨부 + `get_anchor` 도구로 배선(typecheck·테스트·빌드 green, 머지).
- **A/B 검증 2라운드(블라인드 채점):**
  - **라운드1 — 건강 fixture(5시나리오×2조건×K=5):** 5/5 무회귀(median 델타 −3~+1). 단 A·B 모두 critical 결함 0인 케이스라 **상방(효용)은 미측정**.
  - **라운드2 — 결함-유발 fixture(6시나리오×2조건×K=5):** 앵커가 노리는 실패(시니어리티 발명·녹아웃 격상·전이 오판·정량 발명·byte 오인·경력 오분류)를 의도적으로 유발. **A(앵커 없음)가 6/6 함정을 trapRate 100%로 방어** → 앵커 lift 0. S5(byte 제한)만 앵커가 EUC-KR 기본값으로 1건을 오도해 **회귀(100%→80%)**.
- **결론: 강한 모델(opus)에서 앵커는 효용 0 + 1건 회귀 → 기각·되돌림.** 약한 연결 모델은 미검증(이론적 여지는 있으나, S5 같은 잘못된 기본값은 약한 모델에서 더 위험).

## 1. 라운드2 결과 (효용 미입증 — 핵심)

- **S5 회귀 상세:** A 5/5는 "1,500 byte"를 보수적으로(한글≈3byte) 처리해 한도 내. B 1건이 앵커의 `default EUC-KR 2byte` 가정을 빌미로 700자(~1,718 UTF-8 byte)를 쓰고 "1,500 이내"로 단정 → 실제 UTF-8 기준 15% 초과·잘림. **fail-safe라면 가장 빡빡한 인코딩(UTF-8 3byte)을 가정해야 하는데, 앵커가 반대 방향 기본값을 줬다.**

## 2. 왜 기각인가

- **opus + 기존 prose가 이미 함정을 100% 방어한다.** EOP §0a(발명 차단)·"우대≠필수"·count_mode 인지·약점문항 정량 면제·경력/경험 기준이 **이미 코퍼스 산문에 있고 모델이 따른다.** 앵커는 같은 지식의 중복이라 한계효용이 0.
- 입증 책임은 **추가하는 쪽**에 있다 → 효용 미입증 + 1건 회귀 + 컨텍스트 블로트(get_playbook 21KB→30KB) → 되돌림.
- **방법론 한계(정직):** A가 천장(trapRate 100%)이라 B 상방을 잴 여지가 없었다. *"현실 난도에서 효용 0"은 입증*됐으나 *"어떤 난도에서도 무효"는 아니다.* 단 "더 어려운 트랩을 찾을 때까지 파기"는 동기 추구 편향 위험이 있어 보류.

## 3. 만약 재시도한다면 (조건)

이 결론을 뒤집으려면 셋 다 필요: ① **약한 연결 모델**(Haiku/mini/Flash 등 — CareerMate의 실제 사용자 모델)에서 검증, ② `charcount-modes` fail-safe를 **UTF-8 3byte(최강)** 로 고친 뒤, ③ A trapRate가 **<100%로 떨어지는 난도**의 결함-유발 시나리오에서만. 그 전까지 앵커는 도입하지 않는다.

## 4. 상태

- **되돌림:** 앵커 자산 8종 + `get_anchor` 도구 + `get_playbook` 첨부 + 권한 동기화(manifest·init.ts·MCP_TOOLS) 전부 revert. typecheck·테스트 green 재확인.
- **원천 데이터:** 리서치 워크플로우 산출물 + A/B 2라운드(101·121 에이전트) 집계는 세션 스크래치에 보존. 정본 결론은 본 문서.
