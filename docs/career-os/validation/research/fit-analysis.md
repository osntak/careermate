# RESEARCH — 적합도 판정 (Fit Analysis)

> 출처 등급: [1차]=공식/권위, [2차]=평판 HR/ATS 매체, [참고]=커뮤니티/일화.

## A. 전문가 원칙 (실제 리크루터의 사고 모델)
- **P1 — 적합도는 스킬·학력이 아니라 '관련 경험'으로 먼저 결정된다.** SHRM 리크루터 설문: 초기 스크리닝에서 경력연수(84%)·경험유형(83%)이 자격증(37%)·학력(35%)을 압도. 스킬은 경험 부족의 대체재로 인정되지 않음(스킬 평가는 *나중* 단계). → AI는 입증된 관련경험을 척추로, 키워드 매치는 보조 증거로. [2차] https://www.shrm.org/topics-tools/news/talent-acquisition/recruiters-say-experience-top-factor-applicant-evaluation
- **P2 — 채점 전에 MUST(녹아웃) vs NICE(우대)를 분리.** 안 하면 평가자마다 기준이 달라짐. 진짜 녹아웃 = "없으면 직무 수행/법적 자격 불가". 테스트: "그게 없어도 탁월한 지원자면 뽑겠다" → 녹아웃 아님(선호). 녹아웃은 ~3-5개 상한. [2차] https://foundire.com/blog/how-to-design-perfect-knockout-criteria/ , https://www.ihire.com/resourcecenter/employer/pages/candidate-requirements-must-haves-vs-nice-to-haves
- **P3 — 녹아웃은 이진·객관, 우대는 그라데이션.** 녹아웃 표현은 검증가능 Yes/No("PMP 보유", "취업 자격", "5년 이상 X 경력")여야지 주관적("엑셀 잘함") 금지. [2차] foundire, eddy.com
- **P4 — 전이가능 스킬은 '증거'로만 인정, 자기주장 금지.** 기준은 행동 증거(실제로 했는가)이지 능력 주장 아님. 모든 주장 역량을 구체 산출물(배포 프로젝트·지표·맥락 내 도구)에 매핑. [2차] https://hiring.monster.com/resources/recruiting-strategies/talent-acquisition/crossover-skills/
- **P5 — 과적합은 탈락이 아니라 리스크.** 리크루터는 과소자격을 과적합보다 2배 더 기피; 과적합은 보통 우위. 진짜 우려는 retention/이탈이지 역량 아님 → '리스크+완화질문'으로, 자동탈락 금지. [2차] SHRM; https://lucasjamestalent.com/is-overqualification-just-as-problematic-as-underqualification/
- **P6 — 구조화·증거 앵커 채점이 인상 채점을 이긴다.** 구조화 면접 예측타당도 ~.51(단일 최고). 핵심은 주관 인상을 역량별 1-5 앵커 척도의 문서화된 행동증거로 대체하는 것. AI도 요건별 증거 인용+등급, 전역 직감점수 금지. [1차/2차] https://rework.withgoogle.com/intl/en/guides/a-guide-to-structured-interviewing-for-better-hiring-practices ; https://www.pin.com/blog/structured-interviews-guide/

## B. 적합도 채점 방법론 (실행 가능한 단계)
1. **JD를 원자 요건으로 분해** → 3버킷: 필수/MUST(녹아웃)·우대/PREFERRED(가점)·맥락/CONTEXT(직무·환경, 관련성 판단용). 각 요건은 단일 검증가능 주장.
2. **녹아웃 먼저 게이트.** 각 MUST = Yes/No/Unknown. 하드 MUST 미충족(법적자격·필수자격증·진짜 시니어의 "5년 이상") → 나머지와 무관 FAIL/낮음, 녹아웃 명시. 녹아웃 Unknown → 통과불가, blocking missing-info.
3. **요건별 증거 매핑(비-순진 채점의 핵심).** 각 요건 ← 정확한 resume 증거(프로젝트·역할·지표·맥락 내 도구) → 강도: Strong(했음+성과/지표)/Moderate(인접·전이, P4)/Weak(언급, 증거無)/None. **키워드만 있고 행동증거 없으면 Weak.**
4. **가중 채점(평균 아님).** MUST는 게이트(평균 안 들어감). 충족분 중 관련경험증거 > 맥락 내 하드스킬 > 자격 > 우대 가중. 우대는 상방만, 부재는 무감점. 밴드(높음/중간/낮음 또는 0-100)를 추론과 함께 노출.
5. **리스크/레드플래그를 점수와 분리.** 과적합(P5)·미설명 공백기·짧은 근속/잡호핑·스코프미스매치(IC가 리드 지원, 리드 증거無)·도메인미스매치 → 면접 확인 항목이지 감점 아님(진짜 녹아웃 제외).
6. **미싱인포 정직 선언.** resume 침묵 요건 = "확인 필요"(가정된 통과/탈락 금지). 녹아웃급 미상은 verdict 보류, 우대급 미상은 신뢰도만 하향.

## C. ATS 현실 & 신화 (순진한 파서 흉내 금지)
- ATS는 검색/랭크/저장 도구, 거부는 사람이 함. ~99.7% 리크루터가 키워드로 정렬/우선순위 → 위험은 '낮은 랭킹'이지 자동탈락 아님. 커버리지는 *순서*용. [2차] https://blog.theinterviewguys.com/what-ats-looks-for-in-resumes/
- 키워드 매치 ≠ 적합. Jobscan 권장 ~75-80% "스위트스폿"이며 "스터핑 없이 75% 초과 어렵다"고 경고 → **높은 커버리지를 높은 적합으로 등치 금지**(과최적화=게이밍 신호). [2차] https://www.jobscan.co/blog/what-jobscan-match-rate-should-i-aim-for/
- 리터럴 ATS는 동의어 못 함("Adobe Creative Suite"≠"Creative Cloud"). **AI가 메우는 지점**: 동의어·약어·등가 도구 매핑. [2차] interviewguys
- 신화: "낮은 키워드=탈락"(거짓), "매치 카운트로 적합%"(거짓), "높을수록 좋다"(거짓, >75-80%=스터핑), "ATS가 등가어 이해"(거짓).

## D. 한국 시장 특수
- **자격요건/필수/우대.** 자격요건=자격 우산(100% 아니어도 경쟁력 가능). 안의 **필수**(자격증·어학·법적)가 진짜 녹아웃. **우대**=가점/타이브레이커. 단 **우대가 사실상 필수**일 수 있음(시니어/산업·반복·강조 시 격상). [2차] https://www.haijob.co.kr/blog/job-posting-analysis-guide/
- **JD를 리크루터처럼 읽기:** 반복 단어=핵심역량 강조 / 숫자("월 100건","5년 이상")=기대 수준·업무량(정량 최소치는 종종 녹아웃) / 명명 도구(SQL·GA4·Python)=직접 핸즈온 / 동사 시니어리티(신입 지원·수행·참여·작성 → mid 운영·분석·기획·관리 → senior 전략수립·리드·의사결정)로 스코프 미스매치 탐지. [2차] https://blog.viewinterhr.com/guide-jd-case/
- **신입 vs 경력:** 경력=도구+지표 → 즉시전력, 입증 delivery로 평가. 신입=기초·잠재력·직무관련 교육/경험으로 평가, **경력연수 녹아웃 적용 금지.**
- **공채 vs 수시.** 공채=표준화/NCS·코호트 스크리닝. 수시=JD 주도, 글로벌 must/nice에 가깝고 비법적 요건 유연.
- **NCS/능력중심·블라인드.** 직무기술서의 능력단위(지식·기술·태도·경험)에 직접 증거 매핑. 블라인드=출신·학력·외모 배제, 직무능력만. **학교/지역/외모 신호 사용 금지.** [1차] https://www.ncs.go.kr/ , 공정채용 가이드북 https://m.ncs.go.kr/blind/rh13/bbs_lib_list.do?libDstinCd=19
- **법적 가드레일(채용절차법 제4조의3).** 직무무관 개인정보(용모·키·체중·출신지역·혼인·재산·가족 학력/직업/재산) 수집/요구 위법(과태료 ≤5백만). 거짓 채용광고 금지. AI는 이를 적합 요소로 쓰지 말고, JD가 요구하면 플래그. [1차] https://www.law.go.kr/lsInfoP.do?lsiSeq=208371

## E. 적합도 출력 계약 (전문가가 신뢰할 형태 — 전부 포함)
1. 종합 점수+밴드(한 줄 근거, MUST+가중 증거 기반; 키워드% 아님)
2. 녹아웃 패널(각 MUST: 충족/미충족/확인필요, 실패·미상 명시)
3. 요건별 증거 표(요건→증거 인용/위치→강도→전이크레딧 주석)
4. 갭(필수-부재/약한 항목, 임팩트순)
5. 리스크/레드플래그(과적합·근속·스코프·도메인 → 면접 확인 항목)
6. 미싱인포 정직(녹아웃급 미상=blocking)
7. 신뢰도(미상 많을수록 하향)

## F. 리뷰어 채점 루브릭 (AI 적합도 분석 판정 PASS/FAIL)
**즉시 FAIL(얕음) — 하나라도:** 증거 매핑 없는 키워드 카운트 fit% / MUST·NICE 미분리 또는 우대로 자동탈락 / 높은 커버리지=높은 적합 / 증거 환각·미상 가정 / 학력·출신·나이·외모 등 금지신호 사용 / 과적합 자동탈락·신입에 경력녹아웃 / 요건별 증거 없는 인상·단일 직감점수.
**PASS(전문가) — 전부:** JD 원자분해+버킷 / 녹아웃 선게이트+명시 / 요건별 증거강도+전이근거(주장 아닌 행동증거) / 관련경험 가중·우대 상방만 / 리스크 분리 / 미싱인포 정직+신뢰도 조정 / 한국 관행(필수·우대·신입/경력·NCS·금지신호) 준수.
**탁월(가점):** ATS가 놓치는 동의어/도구 등가 포착 / 사실상 필수 우대 식별 / 갭·리스크별 구체 next action.

## 충돌/긴장 메모
- 우대=선택(교과서) vs 사실상 필수(실무): 기본은 가점, 반복·강조·시니어면 격상.
- 스킬기반 vs 경험우선: 스크리닝 단계(=AI 동작 지점)에선 경험 우선·스킬은 증거.
- Jobscan 75-80% 목표 vs ">75%는 스터핑" 경고 → 커버리지는 순서 휴리스틱이지 적합 측정 아님.
