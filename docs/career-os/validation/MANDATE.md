# MANDATE — Career-OS 전문가화 + 3-LLM 교차검증 (절대 잊지 말 것)

> 이 문서는 사용자의 핵심 지시를 보존한다. 컨텍스트가 압축되어도 이 파일과
> [PROTOCOL.md](PROTOCOL.md)를 다시 읽고 이어서 진행한다. **목적에 도달하기 전까지 멈추지 않는다.**

## 한 줄 목적
CareerMate에서 **외부 LLM이 사고(思考)를 담당하는 모든 기능**을, 평범한 LLM이 아니라
**아주 세밀한 도메인 전문가**처럼 수행하게 만드는 실행형 지침(가이드라인)을 만들고,
그 지침을 **Claude·Codex·Gemini 3-LLM 교차검증**으로 만장일치 "완벽" 판정이 날 때까지 단단하게 다듬는다.

## LLM이 사고하는 5대 기능 (전부 전문가화 대상)
1. **적합도 판정** (fit / suitability analysis)
2. **공고 분석·분류** (job posting analysis & classification)
3. **기본 정보 채우기** (profile / 경력·스킬 구조화 추출)
4. **자기소개서 작성** (cover letter / 자소서) — *합격 예시 패턴 분석 포함*
5. **면접 준비** (interview prep — 인성/행동 + 직무/기술)

## 반드시 지킬 원칙
- **검증된 정보만.** 대학 커리어센터·공식 기관·평판 있는 채용/ATS 자료를 1순위로. 커뮤니티·블로그는
  보조이며 일화적임을 명시. 아무 정보나 쓰지 말 것. 헷갈리면 리서치로 확인.
- **시장:** 한국 중심(공채/수시·문항형 자소서·NCS 블라인드·국내 면접) + 영문/글로벌 보조.
- **CareerMate 내부에는 LLM이 없다.** 두뇌=외부 AI, CareerMate=지식·실행 프로토콜 제공.
- 결과물은 결론·근거·리스크·추천·액션·신뢰도·다음단계 형태(사용자 대면). 내부 라우트/raw 지식 노출 금지.

## 검증 루프 (사용자가 정의한 방식)
- **지침이 검증 대상이다.** 산출물은 그 지침의 테스트 케이스다.
- Claude·Codex·Gemini가 **각자** 현재 지침에 맞춰 기능별 산출물을 **여러 개** 생성한다(generator).
- 각 산출물을 **다른 LLM들**이 **새 세션**에서 평가·채점한다(cross-evaluator). 점수가 낮으면 그 결함이
  지침의 약점을 드러낸다 → 지침을 고친다 → 재생성 → 재채점.
- **목표:** 최소 **10개 이상**의 산출물이 3-LLM **만장일치 "완벽"** 판정. 자소서 비중을 높이되 5개 기능에 분포.
  그리고 **지침 문서 자체**도 3-LLM 리뷰를 통과.
- 결과가 바뀔 때마다 새 세션/서브에이전트로 재검증. 만장일치 전까지 구조·지침을 계속 수정.

  — 판정을 큐에 남기고 진행, 최종 사인오프 전 Gemini 통과를 요구.

## 현황·한계 (2026-06-18, 라운드 6 종료) — 발견사실은 [README.md](README.md) 참조
- **실제 달성치는 "2-vendor 교차검증 수렴"이다 — "3-LLM 만장일치"가 아니다.** 돌아간 평가자는 claude 계열 + codex 2-vendor뿐이며 **Gemini는 부재**(rate-limit). 위 "한 줄 목적"의 3-LLM은 *지향*이고, **현재 상태는 2-vendor로 표기**한다. 3-vendor 확장은 후속 과제.
- **평가자 vendor 편향 확인:** claude 평가자가 codex보다 체계적으로 후하다(평균 84.3 vs 77.4, 동일 산출물 36:13 한 방향). → **최종 게이트는 cross-vendor(codex) 기준. claude 단독 자기평가로 "완벽" 사인오프 금지.** (근거·calib 변별 = [README.md](README.md) 발견사실)
