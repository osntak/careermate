# 포지셔닝·메시징 지침 (Positioning & Messaging)

> 경쟁분석(research_0003/0004) 종합에서 도출한 **메시징 결정의 단일 출처**. 랜딩/README/스토어 카피를 쓰거나 고칠 때 이 문서를 기준으로 한다. 사실·수치는 경쟁분석 노트 참조.

## 1. 한 줄 포지셔닝

> **"네 커리어 데이터, 네 컴퓨터에. 네 AI가 그 데이터로 일한다."**
> (로컬 전용 + BYOA + LLM 비내장 + 영속 커리어 저장소 — 경쟁사 누구도 정확히 차지하지 않은 빈칸)

## 2. 리드 원칙 — 아키텍처가 아니라 편익으로

- **하지 말 것:** "MCP-first"를 헤드라인에 두기. MCP는 niche 개념이고 "ChatGPT 안 써요"가 다수다.
- **할 것:** 편익을 먼저("로컬에 안전하게 + 내 AI가 내 데이터로 작성"), MCP·stdio는 각주로.
- 근거: 커뮤니티 조사 — local-first 프레이밍이 클라우드보다 일관되게 우위(r/LocalLLaMA #1 동기=프라이버시).

## 3. 증폭할 4대 차별점 (검증된 시장 교집합)

1. **데이터가 PC를 떠나지 않음** — 경쟁사 전원 클라우드. "내 자소서가 노트북을 떠난 적이 없다."
2. **BYOA(네 AI를 가져와라)** — "ChatGPT는 너를 모르지만, 내 AI는 내 커리어 데이터로 쓴다."
3. **반(反) 스프레이** — 핏 분석 → 검토 → 의도적 지원. auto-apply 자멸 시장의 반대편.
4. **AI-tell 제거 + 자기복제 방지 + AI 탐지 리스크 점검** — generic AI 자소서 즉시 발각(HR 74% 식별)의 정면 해법.

## 4. career-ops(56k★) 대비 포지션

- 사실: `santifer/career-ops` = Claude Code 기반 **무상태 구직 스킬**(스프린트 도구), 56,279★, 포크 생태계 형성.
- 프레이밍: **"구직 스프린트 도구(career-ops, 무상태) vs 영속 커리어 서랍장(CareerMate)."**
- 공존 메시지: career-ops 같은 스킬형 도구가 CareerMate MCP를 *데이터 저장소로* 쓸 수 있다(위협을 우군화). 적대적 비교 금지.

## 5. ATS 미신에 대한 정직한 입장

- 사실: "ATS가 이력서 75% 자동탈락"은 2012년 폐업사(Preptel) 마케팅발 미신. 실제 90~95%는 사람이 본다.
- 입장: 경쟁사처럼 **가짜 "ATS 점수"를 흉내내지 않는다.** 대신 "ATS 미신 정정 + 핵심은 사람 가독성·정합성"으로 정직하게 차별화. (ats-compat verifier는 *실재하는* 기계 파싱 호환만 본다 — 점수 마케팅 아님.)

## 6. 절대 하지 말 것 (안티-포지셔닝)

- **❌ auto-apply/대량지원 기능** — 회신율 ~2%로 자멸 중, HR이 "건너뛸 신호"로 인식. 반-스프레이 구조가 자산.
- **❌ 자체 잡보드 스크레이퍼 탑재** — 사람인 판례(2017다224395) 위험. 발견은 AI 에이전트·동반 MCP에 위임(→ COMPANION_TOOLS.md).
- **❌ 타사 합격자소서 DB 복제** — 법적 리스크. 사용자 직접 저장 "참고문 노트"로.
- **❌ "GPT킬러 우회" 마케팅** — 기만이고 실패 전략. 진정성·면접 방어가능성으로 프레임(→ ai-detection-risk verifier).

## 7. 알려진 리스크와 대응

- **설치 마찰(최대 리스크)** — 비개발자가 설치에서 이탈. .mcpb·Claude "Code 탭" 경로 최단화가 임계경로(진행 중).
- **이름 충돌** — `Careerflow.ai`(클라우드 SaaS, 2M+ 유저)와 혼동 가능. 제품명은 "CareerMate"로 일관(✅ npm·공개 repo=소문자 careermate). 작업 폴더명 `CareerFlow`는 로컬 한정이라 무방하나, 신규 공개 표면엔 "CareerMate" 고정.
